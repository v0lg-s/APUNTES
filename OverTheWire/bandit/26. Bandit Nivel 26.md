En bandit 25 nos dan una llave privada para iniciar sesión en *bandit 26*, sin embargo, la Shell por defecto de *bandit 26* fue cambiada.

Revisé qué shell tenía bandit 26 en el archivo `/etc/passwd`.

```bash
cat /etc/passswd | grep bandit26
```

El resultado es un script en la ruta: `/usr/bin/showtext`

Al revisar su contenido:

```bash
cat /usr/bin/showtext
```

El script contiene:

```bash
#!/bin/sh

export TERM=linux

exec more ~/text.txt
exit 0
```

Lo que hace es ejecutar el comando "*more*" para mostrar el contenido de `text.txt`. Después de ejecutar esta línea ejecuta `exit 0`. Lo que hace que la terminal se cierre, por lo tanto la conexión se cierra.

Lo que toca hacer es buscar la manera de que no se deje de ejecutar el comando **"more"**, este comando tiene un modo interactivo, al que se ingresa cuando el texto que se quiere mostrar por pantalla no cabe en toda la terminal.

En este modo podemos abrir el editor de texto *vi*, el archivo no tiene permisos de escritura habilitados para el usuario *bandit 26* por lo que no podemos hacer se ejecute una Shell desde la ejecución del texto. Podemos realizarlo mediante los comandos del editor de texto *vim*.

1. Primero debemos cambiar la Shell que usará *vim* para interpretar los comandos que enviemos.

```bash
:set shell=/bin/bash
```

2. Después de hacer esto podemos ejecutar comandos de bash desde la interfaz de comandos de *vim* usando el símbolo **!**

```bash
:!bash
```

Así spawneo una Shell bash.
