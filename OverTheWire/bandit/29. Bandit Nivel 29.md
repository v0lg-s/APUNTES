Hay tres maneras de recuperar el README de esos commits.
1. **`GIT-RESTORE`** Restaura un archivo o directorio de un commit específico.
2. **`GIT-CHECKOUT`** Permite mover el puntero HEAD a un commit especificado lo que me permite revisar lo que había en ese commit.
3. **`GIT-LOG -P`** Muestra los commits con los cambios realizados.


```bash
git restore --source 3621de89d8eac9d3b64302bfb2dc67e9a566decd README.md
```

- Tras source indicamos el id del commit y después lo que queremos recuperar, en este caso, `README.md`.
- Actualiza nuestra copia local. (Tenemos a README con la contraseña localmente)

```bash
git checkout 3621de89d8eac9d3b64302bfb2dc67e9a566decd
```

- Visualizamos localmente lo que había en este commit, muy parecido a `restore`. Cambiamos de rama.

```bash
git log -p
```

- Nos muestra un resumen de los cambios realizados en cada commit.
