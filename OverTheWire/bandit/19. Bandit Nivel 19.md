La flag está en un archivo llamado *'readme'*, sin embargo, el archivo *'.bashrc'* fue modificado para hacer que cuando iniciemos sesión con SSH nos expulse del sistema.

El archivo *'.bashrc'* es un script que se ejecuta cada vez que se inicia una nueva sesión de Bash. Este archivo configura el entorno de Bash, define las variables de entorno, funciones, alias y ejecuta comandos.

El sistema primero lee el archivo `/etc/bashrc o /etc/bash.bashrc` (según que distribución utilicemos) y luego el archivo `~/.bashrc` del usuario que ejecuta Bash.

**¿Cómo entrar?**

1. Durante el inicio de sesión, tras introducir la contraseña hay un margen de tiempo que nos permitirá ejecutar comandos antes de que el sistema ejecute el archivo *'.bashrc'*.

	- Al iniciar sesión con el comando `ssh bandit18@host -p 2220` podemos añadir un comando cualquiera a la derecha.
	- Este comando se ejecutara una vez se inicie sesión, justo antes de que se ejecute el archivo *'.bashrc'*.
	
	Lo que tenemos que pensar, es qué comando usar.
	
	**Hay varias opciones:**
	
	- **sh:** este comando abre una Shell básica (sh) que en muchos sistemas no ejecuta .bashrc.
	- **bash --noprofile:** el comando Bash invoca una terminal, la opción no profile evita que se lean archivos de inicio del perfil.
	- **bash --norc:** se invoca una terminal sin leer ni ejecutar *.bashrc*. *--norc* (no run commands).

```bash
ssh bandit18@host -p 2220 bash --norc
```

2. Una vez ejecutado el comando e introducida la contraseña ya podemos acceder al sistema, sin embargo no se carga una terminal con elementos visuales, simplemente está la entrada de comandos en blanco. Pero ya podemos ejecutar comandos.

```bash
cat readme
```
