Cuando entramos al nivel 32 la Shell por defecto de este usuario está configurada como: `/home/bandit32/uppershell`. Este script convierte todos los comandos que enviamos por la terminal a mayúsculas. Por lo que `echo` pasa a ser `ECHO` y la Shell no lo puede interpretar.

Revisando las variables de entorno predefinidas como `HOME` o `PWD` me di cuenta que el sistema aún las mantenía y me mostraba sus valores. Por ejemplo:

```bash
$SHELL # Estaba asignada como /home/bandit32/uppershell
```

Intenté de muchas maneras cambiarla. Como `shell=/bin/bash` pero la terminal lo convertía en: `shell=/BIN/BASH` y no reconocía la ruta.

Revisando el manual encontré la variable de entorno o posición de parámetro `$0`, el contenido de esta variable depende del entorno en que se esté. Por ejemplo, si se invoca desde dentro de un script, la variable `$0` contendrá el nombre del script:

```bash
scriptDePrueba.sh

#!/bin/bash

echo "Prueba $0"
```

La salida será: `Prueba ./scriptDePrueba.sh`

Por otro lado, si se ejecuta en una terminal muestra el nombre del intérprete de comandos actual.

- Si estamos en bash: `$0=bash`
- Si estamos en zsh: `$0=-zsh`
- Si estamos en sh: `$0=sh`

**Diferencia entre `$0` y `SHELL`:** La variable `$SHELL` indica la Shell predeterminada en el archivo `passwd`. La variable `$0` indica la Shell o script en ejecución en el momento.

Por lo que si ejecutamos en la terminal de bandit 32:

```bash
$0
```

La salida será `sh` y la terminal lo interpretará como una invocación. Y nos dará una nueva Shell con el interprete sh.
