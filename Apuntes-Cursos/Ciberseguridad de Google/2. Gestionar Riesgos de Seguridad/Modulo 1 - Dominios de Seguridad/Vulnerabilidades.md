Una vulnerabilidad es una debilidad que puede ser aprovechada o explotada por una amenaza.
- *Es necesario que exista tanto una [[Riesgos y amenazas#Amenaza|amenaza]]  como una vulnerabilidad, para que exista un [[Riesgos y amenazas#Riesgos|riesgo]].*

# *Ejemplos:* 
En general las vulnerabilidades se pueden dar por:
- Firewalls, software o aplicaciones desactualizadas.
- Contraseñas débiles.
- Datos confidenciales desprotegidos.
- Incluso las personas pueden considerarse como una vulnerabilidad. (Sus acciones pueden afectar significativamente la seguridad de la red interna de una organización. Clientes, empleados o proveedores)
	- Para evitar esto los profesionales de seguridad podemos capacitar a las personas que desconocen del tema, por ejemplo, cómo identificar un correo electrónico de suplantación de identidad.
	
Entre otras específicas como:
## ProxyLogon
- Vulnerabilidad de autenticación previa que afecta al servidor Microsoft Exchange. Un atacante puede autenticarse para implementar código malicioso desde una ubicación remota.
## ZeroLogon
- Vulnerabilidad en el protocolo de autenticación Netlogon de Microsoft.
## Log4Shell
- Permite a los atacantes ejecutar código Java en la computadora de otra persona o filtrar información confidencial.
## PetitPotam
- Afecta el gestor de redes de área local (LAN) de nueva tecnología de Windows (NTLM). Permite a un atacante en LAN iniciar una solicitud de autenticación.
## Falsificación de peticiones del lado del servidor
- Permite a los atacantes manipular una aplicación del lado del servidor para que acceda y actualice los recursos del backend.
