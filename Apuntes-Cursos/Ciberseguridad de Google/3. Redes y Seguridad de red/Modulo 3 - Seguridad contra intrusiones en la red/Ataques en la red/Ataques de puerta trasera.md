Las puertas traseras son puntos débiles dejados intencionadamente por programadores o administradores de sistemas y redes que eluden los mecanismos normales de control de acceso. Ayudan a llevar a cabo la solución de problemas o tareas administrativas. 

Los atacantes tras haber comprometido un sistema pueden instalar puertas traseras para asegurarse un acceso persistente al sistema. Un atacante tras entrar por una puerta trasera puede:
- Instalar software malicioso.
- Realizar un ataque de denegación del servicio (DoS).
- Robar información privada o cambiar otras configuraciones de seguridad que dejen el sistema vulnerable a otros ataques.
