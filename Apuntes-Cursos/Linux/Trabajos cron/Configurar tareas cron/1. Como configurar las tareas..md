Las tareas *cron* se configuran en un archivo del sistema llamado **'crontab'** (Cron Table). 
- Cada usuario cuenta con un archivo **Crontab**.
- Existe un archivo **Crontab** global.
- Este archivo global existe en: **`/etc/crontab`**
- El archivo de cada usuario existe en: **`/var/spool/cron/crontabs/`**
# Archivo crontab
El archivo crontab tiene una estructura específica para programar comandos.

```scss
* * * * * comando_a_ejecutar
- - - - -
| | | | |
| | | | +---- Dia de la semana (0 - 7) [0 y 7 representan domingo]
| | | +------ Mes (1 - 12)
| | +-------- Dia del mes (1 - 31)
| +---------- Hora (0 - 23)
+------------ Minuto (0 - 59)
```

La diferencia con el archivo **crontab** global es que el global tiene un campo adicional dónde se especifica el usuario con el que se desea ejecutar el comando.

- **Minuto:** En qué minuto de la hora se ejecuta el comando (0-59).
- **Hora:** En qué hora se ejecuta el comando (0-23).
- **Día del mes:** En qué día del mes se ejecuta (1-31).
- **Mes:** En qué mes se ejecuta (1-12).
- **Día de la semana:** Qué día de la semana.

# Considerar
**Evaluación día del mes y día de la semana**
- El campo del día del mes y el día de la semana se evalúan con "O" en vez de "Y", es decir, si ambos tienen valores, el cron se ejecutará si cualquiera de ellos coincide.
**Ejemplo:**

```plaintext
0 0 1 * 0 /ruta/script.sh
```

Se ejecutará el primer día de cada mes o cada domingo.

**Rutas absolutas**
- Usar siempre las rutas absolutas a comandos y scripts.
	- Es útil recordar que todos los comandos a nivel de sistema se encuentran en la ruta */usr/bin/...*
**Ejemplo:**

```plaintext
* * * * * /usr/bin/echo "Hola mundo" 
```

**Redirigir salidas**
- Es buena práctica definir en dónde se registrarán las salidas y errores de los comandos o scripts.

**Ejemplo:**

```plaintext
* * * * * /usr/bin/echo "Hola mundo" >> /ruta/log.txt 2>&1
```
