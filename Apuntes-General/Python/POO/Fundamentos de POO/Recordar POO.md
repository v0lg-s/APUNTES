La programación Orientada a Objetos es un paradigma de programación, es decir, un modelo o estilo de programación). Es el sucesor de la programación procedimental, que se basa en la ejecución de una secuencia de líneas una tras otra.

Este tipo de programación es útil aplicado a proyectos grandes y complejos.

Se basa en entidades llamadas objetos que se implementan a partir de clases.

# Clase
Una clase es una plantilla. Esta define de manera genérica cómo van a ser los objetos de un determinado tipo. 

> Una clase es como una categoría.
> Puede ir desde lo más genérico, hasta clases especializadas.
> Tiene una serie de atributos (propiedades) y una serie de comportamientos (métodos)
> **También se puede definir como:** Un conjunto de objetos

*Ejemplo:*

| Clase: Vehiculo |
| --------------- |

| --------------------------- |

# Objeto
Un objeto es un 'ser' perteneciente a una clase.

> Un objeto que pertenece a una clase específica, pertenece a todas las superclases al tiempo.
> Es una implementación de los requisitos, rasgos y cualidades asignados a una clase específica.

## Atributos de un objeto

- Un objeto tiene un nombre que lo identifica de forma exclusiva en su namespace.
- Un objeto tiene un conjunto de propiedades individuales que lo hacen original.
- Un objeto tiene un conjunto de habilidades para realizar actividades específicas.

*Ejemplo*
Rodolfo es un gato grande que duerme todo el día.  
*Nombre del objeto* = Rodolfo  
*Class* = Gato  
*Propiedad* = Tamaño (Grande)  
*Actividad* = Dormir (Todo el día)
