El objeto `IOError` tiene una propiedad llamada `errno` (error number/número de error) y se puede acceder de la siguiente manera:

```python
try:
	# operaciones con streams
except IOError as exc:
	print(exc.errno)
```

Constantes del atributo `errno`.

- `errno.EACCES` $\rightarrow$ Permiso denegado. (*Ej*. Abrir archivo con atributos de solo lectura para escritura)
- `errno.EBADF` $\rightarrow$ Número de archivo incorrecto. (*Ej*. Operar stream sin abrirlo)
- `errno.EEXIST` $\rightarrow$ Archivo existente. 
- `errno.EFBIG` $\rightarrow$ Archivo demasiado grande.
- `errno.EISDIR` $\rightarrow$ Es un directorio.
- `errno.EMFILE` $\rightarrow$ Demasiados archivos abiertos.
- `errno.ENOENT` $\rightarrow$ Archivo o directorio no existe.
- `errno.ENOSPC` $\rightarrow$ No queda espacio en el dispositivo.

Se pueden usar de la siguiente manera, aunque sea la menos efectiva.

```python
import errno

try:
    s = open("c:/users/user/Desktop/file.txt", "rt")
    # El procesamiento va aquí.
    s.close()
except Exception as exc:
    if exc.errno == errno.ENOENT:
        print("El archivo no existe.")
    elif exc.errno == errno.EMFILE:
        print("Demasiados archivos abiertos.")
    else:
        print("El numero del error es:", exc.errno)
```

# Función strerror()

Proviene del modulo `os` y espera un argumento: **un número de error**. Proporciona una cadena que describe el significado del error,

```python
from os import strerror

try:
    s = open("c:/users/user/Desktop/file.txt", "rt")
    # El procesamiento va aquí.
    s.close()
except Exception as exc:
    print("El archivo no pudo ser abierto:", strerror(exc.errno))
```

```mathematica
El archivo no pudo ser abierto: No such file or directory
```
