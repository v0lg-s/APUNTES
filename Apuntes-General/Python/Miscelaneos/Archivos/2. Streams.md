Un stream se comporta como un disco duro:
# Lectura

- Cuando se lee de un stream, un cabezal virtual apunta a una posición. Cada vez que se leen datos, ese cabezal avanza tantos bytes como se hayan leído, lo que ya pasó por el cabezal no se vuelve a leer a menos que se reinicie el stream.

```mathematica
Stream:  [ A ][ B ][ C ][ D ][ E ][ F ]
           ^
         puntero
```

**Se leen 2 bytes**

```mathematica
leido:   [ A ][ B ]

Stream:  [ A ][ B ][ C ][ D ][ E ][ F ]
                     ^
                   puntero
```

El puntero **avanza automáticamente** dos posiciones.

# Escritura

En la escritura el mismo cabezal/puntero se mueve a lo largo del stream registrando los datos de la memoria.

**Estado inicial (stream vacío)**
```mathematica
Stream:  [  ][  ][  ][  ][  ][  ]
           ^
         puntero (posición de escritura)
```

**Escribir 2 bytes**

```mathematica
escrito: [ A ][ B ]

Stream:  [ A ][ B ][  ][  ][  ][  ]
                     ^
                   puntero
```

# Apertura de stream
La apertura de un flujo no solo está asociada al archivo, también se debe declarar la manera en que se procesará el stream. A esto se le llama **open mode (modo de apertura)**.

Hay dos operaciones básicas a realizar con el stream, **lectura** y **escritura**.
# Modos básicos para abrir un stream

- **Modo Lectura:** Un stream abierto en este modo solo permite operaciones de *lectura*. 
- **Modo Escritura:** Un stream abierto en este modo solo permite operaciones de *escritura*.
- **Modo Actualizar:** En este modo el stream permite *ambas* operaciones.

**Nota:** Cualquier intento por realizar una operación no permitida en alguno de los modos levantará una excepción llamada `UnsupportedOperation`
