# Nombre de los archivos

Los diferentes sistemas operativos tratan a los archivos de diferentes maneras. Por ejemplo,
## Case sensitive
- En sistemas *Unix/Linux* los nombres de archivo distinguen entre mayúsculas y minúsculas.
	- `Nombre_Archivo` $\neq$ `nombre_archivo` $\implies$ 2 archivos distintos.
- En sistemas *Windows* los nombres de archivo no lo hacen.
	- `Nombre_Archivo` $=$  `nombre_archivo` $\implies$ Mismo nombre.

## Separador
- En sistemas *Unix/Linux* los nombres de directorio usan como separador `/` *barra*.
	- `/Home/Descargas/` 
- En sistemas *Windows* el separador es `\` *barra invertida*.
	- `C:\Escritorio\Prueba\`
	- **Nota:** Recordar que para que python interprete la barra invertida se debe usar doble. Carácter de escape. `name = "\\Escritorio\\Prueba\\"`

Python es lo suficientemente inteligente como para poder convertir diagonales en diagonales invertidas cada vez que descubre que el sistema operativo lo requiere.

# Comunicación

Prácticamente ninguno de los lenguajes de programación se comunica con los archivos directamente, lo hacen a través de entidades abstractas que se nombran de manera diferente según el lenguaje. Los más conocidos son *handlers/manejadores* (Una clase de puntero inteligente) o *streams/flujos* (Una especie de canal).

Con los métodos y funciones se pueden realizar ciertas operaciones en el flujo, que afectan los archivos reales.
## Operaciones del flujo

- **Conectar el flujo con el archivo:** Abrir el archivo
- **Desconectar el enlace:** Cerrar el archivo
- **Lectura del flujo:** Porciones de datos del archivo se colocan en un area de memoria que administra el programa, por ejemplo, una variable.
- **Escritura del flujo:** Porciones de datos de la memoria se transfieren al archivo.

La primer operación siempre es `open (abrir)` y la última `close (cerrar)`
