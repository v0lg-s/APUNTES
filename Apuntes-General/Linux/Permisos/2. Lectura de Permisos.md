Los permisos van de izquierda a derecha en pares de tres.

* -*rwx rwx rwx* : 
	* El primer grupo de tres bits indica los permisos que el dueño "[[1. Tipos de permisos y Usuarios#1. Usuario/User|1. Tipos de permisos y Usuarios]]" del archivo tiene, 
	* El segundo grupo de bits indica los permisos que el grupo "[[1. Tipos de permisos y Usuarios#2. Grupo/Group|1. Tipos de permisos y Usuarios]]" especificado tiene sobre el archivo
	* El último grupo de bits indica los permisos que otros "[[1. Tipos de permisos y Usuarios#3. Otros/Other|1. Tipos de permisos y Usuarios]]" usuarios tienen sobre dicho archivo.
* El primer espacio cuando se encuentra una "d" indica que se trata de un directorio.
* Cuando en vez de una de las letras se encuentra un guión, por ejemplo:
	* -rw-r--r--: indica que ese permiso no se encuentra dado. En este caso, "[[1. Tipos de permisos y Usuarios#1. Usuario/User|1. Tipos de permisos y Usuarios]]" no tiene permiso para ejecutar, "[[1. Tipos de permisos y Usuarios#2. Grupo/Group|1. Tipos de permisos y Usuarios]]" no tiene permiso para escribir ni ejecutar al igual que "[[1. Tipos de permisos y Usuarios#3. Otros/Other|1. Tipos de permisos y Usuarios]]".

*Permisos:* 
- r → [[1. Tipos de permisos y Usuarios#1. Read/Leer|read]]
- w → [[1. Tipos de permisos y Usuarios#2. Write/Escribir|write]]
- x → [[1. Tipos de permisos y Usuarios#3. Execute/Ejecutar|execute]]

Se pueden leer como binarios y pasarlos a decimal para usarlos junto al comando [[3. Cambiar Permisos#Comando chmod|chmod]]. Es decir, los siguientes permisos se pueden ver como:

*-rw-r---w-.* → *110 100 010* 

Si pasamos a decimal cada 3 bits obtenemos:

6 4 2
