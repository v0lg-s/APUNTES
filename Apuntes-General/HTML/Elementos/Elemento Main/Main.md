Este elemento se usa para representar el contenido principal del **`<body>`** de un documento o aplicación. 

Representa el contenido central y único que está directamente relacionado con el objetivo principal del documento o aplicación web.

- **Contenido principal:** Abarca la información más relevante de la página.
- **Evita duplicidad:** Solo puede haber **un elemento `<main>`** por documento HTML.
- **Accesibilidad:** Mejora la navegación para lectores de pantalla, permitiendo saltar directamente al contenido principal.
- **SEO (Optimización en buscadores):** Ayuda a los motores de búsqueda a identificar cuál es el contenido más importante de la página.

# En qué casos se usa `<main>`?

1. **Páginas de inicio o landing pages:** Para encapsular contenido central como textos, imágenes o llamados a la acción.
2. **Blogs o artículos:** Rodea el cuerpo principal del artículo.
3. **Aplicaciones web:** Contiene la parte principal interactiva de la app.

# Tener en cuenta
- Solo debe aparecer **una vez** por documento.
- No debe estar anidado dentro de otras etiquetas semánticas como `<header>`, `<footer>` o `<article>`.
- Úsalo para contenido **único y relevante**, no para componentes globales o repetitivos
